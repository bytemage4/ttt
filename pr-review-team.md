1. The adversarial pass (3b-iii) should arguably be a fresh-context subagent. The whole point of "assume a real bug the first pass rationalised as fine" is the opposite prior — but if the same context window did the first pass, it's anchored on its own reasoning. A fresh subagent with an explicit "opposite prior" prompt is much more likely to actually find what the first pass missed. As written, the spec just says "Opposite prior" without specifying the mechanism, which probably means the main agent does it.
2. Comment composition (3d, the new step I added) inherits the same problem. If the same context that ran the review writes the comments, it'll carry over its own framing — including any over-explanation it produced internally. A fresh-context comment-writer that only sees the kept findings from triage would naturally write tighter, more direct comments because it doesn't have the reviewer's internal monologue to lean on.
